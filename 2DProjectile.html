2D Projectile

This is a project I started with my friend during a boring summer period. Our goal was to make a platform game that resembled Hollow Knights style.
This is the only project that I would call a failed project as there but for the projectile physics of main character gun.

Despite it being a failure I really think that this project was essential for me to understand how physicals projects like gravity are implemented in a videogame or in a computer context in general.
What I learned is that all the physics formulas like projectile motions I learned during A-levels where not as useless as I though they were.

As you can see the projectile motion of the projectile follows the form of projectile motion, the green dot are used to aim, they are prediction of the path the ball is going to follow so that the player can take the aim with the mouse cursor.
The velocity of the shot is given by the force applied to the bullet, and the force is increased depending on the distance the cursor has from the player character.

I honestly think this is the only relevant piece of code that makes this project stand out from the others, as I never attempt on using mathematical/physical formula in any of my progrraming before this project. I found it very useful and interesting.

It is worth knowing that before I tried this physical approach, I first was experimentin with parabolas of the form x2+yx+c by using the player position as origin and the constant to find a good projetile. However I had small bugs as the bullet kept following a circular path, therefore I changed the way of watching at the problem and went for a physical approach.
