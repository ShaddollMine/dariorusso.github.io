A-levels RPG

The A-level computer science project consisted of creating the documentation for a code that I had to produce.

The first step was to find a problem that my code would have solved; unfortunately, entertainment was not a good enough reason.
After some brainstorming and online research, I decided to create a videogame that would have helped young GCSE students learning timetables.
After completing the research, I run a survey to GCSE students and teachers to understand my target stakeholders.
From the survey data gathering, I filtered the type of game I had to implement (2D RPG tileset based videogame). Discovering that teachers were worried about not being able to understand student's progress on timetables through the game interface.
For this reason, I decided to start thinking about having a database that stores each student's score and how they got to have that score. Now that I had a database, I needed to create an access point for the teachers to visualize the database. Therefore, I decided to implement an application that allows teachers to generate basic SQL queries, hiding the complexity of SQL.

But now, let's go over the application:

The main menu asks the user to register or login.

The registration part is just a form the student or teacher has to compile. The information gets stored in the database after a small consistency check is performed.

The login asks for a username and password. The login understands though the username if the person logging in is a teacher or a student and dispatches them in the appropriate section.

In the case of teachers, it brings them to a menu where they can select various options to visualize the database.

This is an example of visualizing the database.

In the case of a student, the videogame starts bringing them in the first room where they can find six different entities:

  • A speech NPC
  • A Fight NPC
  • A Challenge NPC
  • A portal.
  • Attack potion item
  • Health potion item

The speech NPC is simply an NPC that speaks to you if you approach him.

A fight NPC is an NPC that: if approached, asks you to fight him. Accepting this will bring you to the fighting part of the game.

A challenge NPC is an NPC that obligates you to fight him bringing you to the fighting part of the game.

The portal is a way for the user to move between rooms: if it is interacted with, the player is teleported to another map.

The health and attack potions are items that: if interacted with, will be stored in your inventory. These items are useful during the fighting part of the game.

The last feature to talk about is the fighting scene; fighting is based on the Pokemon turn-based RPG game fight; in this fight, you can:

  • Defend yourself from the incoming attack; this will heal you.
  • Attack the enemy; this damages the enemy;
  • Use an item (defense of attack item); this heals or increases your attack damage

After completing all the parts I had to test my code. I produced test cases and run them discovering a couple of bugs. After testing it, I went over the acceptance test and completed my report. All these pieces of information can be found in the report I produced, with a better explanation in all the sections.
